## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Project setup

```bash
$ npm install
```

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

```bash

$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```

# LeetCode Backend Clone

This project is a backend implementation for a LeetCode-like platform, built with NestJS, TypeORM, and GraphQL.

## Entity Structure

### User

- Core user information (username, email, password)
- Profile details (displayName, bio, avatarUrl)
- Role-based permissions (USER, ADMIN, MODERATOR)
- Statistics (reputation, totalSolved, currentStreak, etc.)
- Relationships with other entities (submissions, discussions, etc.)

### Problem

- Problem details (title, description, difficulty)
- Metadata (tags, constraints, examples, hints)
- Statistics (likes, dislikes, submissions, acceptance rate)
- Test cases for validating solutions
- Settings (timeLimit, memoryLimit, isPremium)

### ProblemTestCase

- Input/output pairs for testing submissions
- Associated problem reference

### ProblemList

- User-created collections of problems
- Public/private setting
- Associated user reference

### Submission

- Code submitted by a user
- Language used
- Status (ACCEPTED, WRONG_ANSWER, etc.)
- Performance metrics (runtime, memory)
- References to user and problem

### Contest

- Contest details (title, description)
- Time constraints (startTime, endTime)
- Status (UPCOMING, ONGOING, FINISHED)
- Associated problems
- Registration information

### ContestRegistration

- Links user to a contest
- Tracks score and rank
- Registration timestamp

### Discussion

- Forum post with title and content
- Associated user (author)
- Optional association with a problem
- Voting system (upvotes, downvotes)
- View counter
- Comments

### Comment

- Text content
- Associated discussion
- Associated user (author)
- Nested comments (parentId)
- Voting system (upvotes, downvotes)

### UserSubscription

- Links user to a subscription plan
- Plan type (FREE, PREMIUM)
- Time period (startDate, endDate)
- Settings (isActive, autoRenew)

### Notification

- Message to a user
- Read status
- Optional link to an entity (entityType, entityId)

### NotificationSettings

- User preferences for different notification types
- Email notification preferences

## GraphQL Schema

The project uses code-first GraphQL schema generation with NestJS decorators. The schema is automatically generated based on the entity definitions and resolver implementations.

## Database Configuration

The application uses MySQL database with TypeORM for ORM functionality.

## Running the Application

1. Ensure MySQL is running
2. Update database connection details in app.module.ts if needed
3. Run `npm install` to install dependencies
4. Run `npm run start:dev` to start the application in development mode

## API Documentation

Once running, the GraphQL playground is available at `/graphql` endpoint.
